Getting Started

Follow the steps below for setting up your app.

Install Dependant Libraries
To get the app up and running you'll first need to install all the required libraries. Run npm install in the root of the app directory.
Configure Localhost
You'll want to decide on the hostname you want to use for developing the app locally. This can be set in the package.json file here [app-directory]/package.json. Change this line to match the desired hostname and set an open and available port to use.
"start-secure": "ng serve --host=kini-starter-app.oxil.uk --port=6999 --ssl" You'll also want to make sure you have added this hostname to your local /etc/hosts file.
Set Environment Variables
To get the app to start working and communicating with the backend you will need to configure various URLs. Head to [app-directory]/src/environments and update both the production and dev json files.
Set App Colour Palette
There are two places you need to update the colour palettes - one is for use with the Angular Material library we use for some widgets and components. The other is for the CSS library we use for theming the whole app.
Material colour palette can be found here [app-directory]/src/custom-theme.scss You will then need to update the two palette variables named $palette-primary and $palette-secondary. You can use the following service for generating a full palette based on a primary colour http://mcg.mbitson.com/#!?mcgpalette0=%233f51b5
Tailwind CSS colour palette can be found here [app-directory]/tailwind.config.js You will then need to update the colour objects for textColor, backgroundColor, and borderColor. You can find the full range of CSS classes available for styling the app at https://v2.tailwindcss.com/docs/flex
Start the App
To start the app run npm run start-secure in the root of the app directory.
